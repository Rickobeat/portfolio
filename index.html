M-Pesa Forensic Analysis Suite
Desktop Application Environment Pipeline // System Docs

A desktop application for forensic analysis of M-Pesa statements built for AML detection, credit scoring, and transaction pattern analysis.

What It Does

Ingests M-Pesa PDF statements and turns them into a structured forensic report: transaction reconstruction, loan repayment behavior, and anti-money-laundering (AML) risk signals. Built for cases where a statement needs more scrutiny than a simple balance check loan underwriting, dispute investigation, or compliance review.

Key Features
  • Forensic Reporting - A structured, 15-stage audit trail per statement: subject info → statement authenticity → transaction breakdown → income adjustments → financial ratios → cash-flow volatility → liquidity/dependency → stability indicators → gambling activity → large deposit flags → risk summary → savings behavior → digital credit exposure → final risk/stability rating → AML pattern analysis → credit decision. Each stage renders as its own report card with pass/warn/fail signal coloring.
  • AI-Generated Summary - An integrated analysis layer sends the compiled report to a connected LLM (supports OpenAI, Anthropic, or Cohere-compatible endpoints) with a forensic-auditor system prompt, returning a plain language summary of red flags, tampering signs, AML concerns, and a final credit recommendation with live connection diagnostics for whichever provider is configured.
  • Risk Visualization - A radar-style sonar chart for at a glance risk exposure, plus donut and dual-bar charts for transaction category breakdowns, monthly cash flow, and digital credit exposure.
  • AML Pattern Detection - Automated flagging of suspicious transaction patterns: structuring, fan in/fan out clustering, rapid pass-through of funds, and dormancy spikes.
  • Loan Repayment Ratio (LRR) - A custom metric scoped to overdraft (Fuliza) transactions, measuring repayment consistency relative to usage.
  • Currency Circulation Threat Metrics - Flags cumulative exposure from digital credit provider loans, betting/gambling fund flows, and unusually large cumulative deposits.
  • Statement Integrity Checks - Font fingerprinting analysis to help detect tampered or doctored statements.
  • Fast Search - Trigram indexed, debounced search across large transaction tables with no UI lag.
Design Architecture

Dark terminal/HUD aesthetic frameless window, custom drag/resize handling, unified green cyan color scheme, and animated radar/sonar visual elements. Designed to feel like a tactical analysis console rather than a typical business app but najua hujasoma hiyo yote.

Tech Stack Core

Python · Custom charting and animation layer · Pluggable LLM integration (OpenAI/Anthropic/Cohere-compatible)

Rickobeat Bot
General Purpose Telegram AI Assistant // System Docs

A general purpose conversational AI bot running on Telegram (@cosyspecbot), built for free form chat with real time streaming responses and clean, chat native formatting.

What It Does

Handles any incoming text message, private chat or group mention and streams back an LLM generated reply as it's produced, editing the message live rather than waiting for a full response. Maintains a rolling conversation window per user so replies stay contextually aware without unbounded memory growth.

Key Features
  • Live Token Streaming - Responses are streamed from the model and edited into the chat message in near real time, with a typing indicator kept alive throughout generation.
  • Markdown → Telegram HTML Formatter - A custom converter turns model markdown (headings, bold, inline code, fenced code blocks, tables) into Telegram safe HTML, including fixed-width table rendering and ANSI escape cleanup.
  • Per-User Conversation Memory - A rolling history window keeps recent turns in context per user, resettable on demand via a "new chat" command.
  • Daily Rate Limiting - Per-user daily quota with a decaying pool of varied "quota exhausted" responses before falling back to a hard stop message, plus a usage check command showing remaining requests and reset time.
  • Group-Aware Routing - In group chats, the bot only responds when explicitly mentioned, stripping the mention out of the prompt automatically.
  • Resilient Delivery - Handles Telegram flood control backoff, gracefully falls back to plain text if HTML formatting fails, and automatically chunks long responses across multiple messages.
  • Personality Layer - Non text uploads (photos, documents, voice, video) are declined with a randomized, decaying pool of in-character rebuff messages rather than being processed.
Tech Stack Core

Python · python-telegram-bot · LLM inference · Threaded streaming pipeline over asyncio
